# 4.Github 연결 Tutorial (잘 안될 만한 포인트만)

깃허브에 레포지토리 만들어서 로컬 저장소의 폴더랑 연결할 때, 

```bash
git remote add origin "연결할 본인 깃허브 레포지토리 주소"
git add "연결할 로컬저장소의 파일명"
```

이렇게 하면 처음에

```bash
To push the current branch and set the remote as upstream use git push…어쩌고
```

이런 에러가 뜸

왜 뜬거냐면, Git은 실수로 엉뚱한 곳에 코드를 올리는 것을 방지하기 위해 매우 보수적으로 작동함. 

그래서 "어디로 올릴지 확실하지 않으니, 다음부터는 자동으로 올릴 수 있게 **상류(Upstream)** 관계를 설정해달라"고 요청하는 것

```bash
**git push --set-upstream origin "브랜치명"**
```

이렇게 하면 다음부터 자동으로 오류 없이 업로드 가능

# 6.Git Branch

다 같이 음식 만드는데, 누가 갑자기 후추 들어가면 어떰? 하면서 대뜸 쳐 뿌리면 어떨까

![강호동.png](/assets/img/GB/강호동.png)

아무래도 그러면 쉽지않기 때문에, 일단 혼자 좀 만들어보고 맛을 보여줘야 하지 않을까?

그게 branch를 만드는 이유



<br>
기본으로 일단 main이 메인 branch기 때문에,

```bash
git branch
```

라고 하면 초록색으로

```bash
* main
```

이라고 뜬다.

여기서,

```bash
git branch "새로 만들 브랜치명"
git branch
* main
  "새로 만든 브랜치명"
```

이렇게 새로 만든 브랜치가 뜰 텐데, 

```bash
git branch -m "바꿀 기존 브랜치명" "바꿀 새로운 브랜치명"  #브랜치 이름 바꾸기
```

```bash
git checkout "넘어갈 브랜치명"
```

이렇게 다른 브랜치로 넘어가서 코드를 만들 수 있다.

이때 분기된 브랜치에서 쓴 모든 코드는 다른 브랜치에 전혀 영향을 주지 않는다.

![Nexus.jpg](/assets/img/GB/Nexus.jpg)

(드라마 로키에서 나온 시간 분기점 같은 느낌..)

분기된 브랜치에서 짠 코드를 메인에 병합하고 싶다면,

다시 main으로 돌아와서,

```bash
git checkout main

git merge "병합할 브랜치명"
```

이러면 이제 main에도 분기된 브랜치에 있던 코드가 들어온다.
